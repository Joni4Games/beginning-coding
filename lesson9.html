Objects

An object takes the concept of a function further, allowing you to encapsulate an entire set of functionality into a single reusable place, which can include multiple variables and functions, although when they are part of an object, they are called properties and methods. We've already seen some built in methods and properties being used in our previous code examples. For example, fillRect is a method of any canvas object that draws a rectangle; fillStyle is a property of any canvas that contains the pen colour you've set for the canvas.

In JavaScript, everything is an object, the methods and properties of which depend on the type of object and the context it is being used in.

As another example, when you store a string in a variable, that variable is an object. All string variables automatically have a property called length (amongst others), which contains the number of characters in the string. So in this case:

myName = 'Chris';
myName.length;

myName.length would return 5.

An object is generally meant to be modelling a real life thing in an abstract way, with different features of that thing grouped together and described in code. You can create your own objects; I won't go into this too much because it is quite advanced, but I'll include an example just for completeness' sake.

If you wanted to store information on dogs, you can create a Dog object like so:

var Dog = {};

This is called Object literal notation, and is only one way of creating an object. This is an empty object that we can now add properties and methods into, by using dot notation: 

Dog.legs = 4;
Dog.tail = true;
Dog.food = "meat";
Dog.bark = function() {
  alert("Woof!!");
}

Enter all the code in this section so far into the JS pane on JSBin, then run it. Try typing some commands into the console like this, to see what you get:

Dog;
Dog.food;
Dog.bark();

You can also extend objects to create new objects with more specific attributes. These objects inherit all the properties and methods of their parents. If we wanted to create a more specific class of dog, say a Golden Retriever, we would have to first create a constructor function, which is function containing method and property information to build an object from:

function Dog(breed,coat) {
// It's a normal function, with two arguments passed in for
// more specific information that vary for each sub class 
   this.legs = 4;
   // these are constant for all Dog sub-classes. "this.legs"
   // means "legs is being made a property of the current object,
   // whatever that is"
   this.tail = true;
   this.food = "meat";
   this.bark = function() {
     alert("Woof!!"); 
   };
  
   this.breed = breed;
   // These will vary by sub-class, depending on what values you
   // give the constructor function's arguments when you create
   // each subclass.
   this.coat = coat;
}

Now to create a specific subclass of Dog called Retriever, you do this:

var Retriever = new Dog("Golden Retriever","Golden");

Try putting the above two code blocks into the JavaScript pane of JSBin. Now try typing the following into the console:

Retriever

You'll see that you get an object returned with all the properties that the Dog constructor defines, plus the two specific properties we defined.

You can even add other specific properties using the same dot notation we saw before, to further extend the Retriever class:

Retriever.mood = "Chilled out";

We will leave our explanation of objects there, as they are complicated, and our time is short!

How to go further

I hope this brief exploration of programming has helped give you a rung on the ladder, and inspired you to go and learn more! Next, I would advise you to do the following:

  • Sign up to Codecademy and work through all the JavaScript Fundamentals courses. This is a great way to get the basics into your head.
  • Work through some of the other courses on codecademy such as the Web fundamentals, jQuery, Python and Ruby courses.
  • Have a look Code School's training videos
  • Download and read the rather excellent free book, Eloquent JavaScript.
  • Look at lots of other people's code! Go on to your favourite web sites and view source, go to Github and search for some interesting code.
